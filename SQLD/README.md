### 1. SQL의 정의

* SQL(Structured Query Language)은 데이터베이스를 직접 액세스할 수 있는 언어

* 데이터 정의(Data Definition), 조작(Manipulation), 조작 결과를 적용 혹은 취소(Transaction Control), 접근 제어(Data Control)

<br>

### 2. SQL 개발자(SQLD)의 정의

* SQL 개발자(SQLD, SQL Developer): 데이터베이스와 데이터 모델링 지식을 바탕으로 데이터를 조작하고 추출하는 데 있어서 정확하고 최적의 성능을 발휘하는 SQL을 작성할 수 있는 개발자
* Part 2까지 해당
<br>

### 3. SQL 전문가(SQLP)의 정의

* SQL 전문가(SQLP, SQL Professional): 데이터베이스와 데이터 모델링에 대한 지식을 바탕으로 데이터를 조작하고 추출하는데 있어서 정확하고 최적의 성능을 발휘하는 SQL을 작성, 이를 토대로 SQL을 내포하는 데이터베이스 프로그램이나 응용 소프트웨어의 성능을 최적화하거나, 이러한 성능 최적화를 지원할 수 있는 데이터베이스 개체(뷰, 인덱스 등)의 설계와 구현 등의 직무를 수행하는 전문가를 말한다

* 키워드: 데이터모델링 지식, 최적의 성능을 발휘하는 SQL 작성, 응용 프로그램 성능 최적화, 데이터베이스 개체(뷰, 인덱스) 설계
* 전 Part 해당

# 🎈 SQL 전문가 가이드
## 0. 들어가며
### ✨ Topic. 나는 왜 SQLD를 선택했을까?
* 애플리케이션 개발에 있어 Database 사용이 필연적이다.
* 특히, RDBMS는 무난하게 사용하는데, RDBMS를 이용하기 위해서는 SQL을 알아야 한다.
* 기본적인 CRUD 기능을 위한 기본적인 SQL 작성에 대해서는 잘 알고 있지만, 그것의 **설계나 성능에 대한 확신**이 없다.
* 데이터 모델의 구조, 데이터 모델링 등 데이터 설계를 원리적으로 이해하고 적용할 수 있는 엔지니어가 되고싶다.
* 또한 더 나아가 SQL 인덱스, 옵티마이저, Lock 등 성능 최적화를 적용하고 싶다. 
* 이 자격증으로서 **기술의 관심도**, **계획 및 실천성**, **개발 직무에 대한 나의 철학**을 알리고 싶다

<br>

### ✨ Topic. 나는 왜 SQL 전문가 가이드를 선택했을까?
* 우선, 나는 이론서적 보는 것을 좋아한다.
* **SQL 전문가 가이드**는 K-data 데이터자격검정에서 채택한 공식 지침서이다. => 무엇이 official인지 고민할 필요없다
* 기본적인 SQL 외에 심화된 내용을 공부할 수 있다. => 일반적인 전공서적과는 다르다
